	3. 权限管理命令
		1. chmod
			1. chmod命令
				1. 英文：change the permissions mode of a file
				2. 命令所在路径：/bin/chmod
				3. 执行权限：所有用户
				4. 功能：更改文件权限
			2. 语法：chmod [{ugoa}{+-=}{rwx}] [文件或目录] [mode=421] [文件或目录] -R [递归修改]
				1. {ugoa}：所有者，所属组，其他人，所有人
				2. {+-=}：添加，删除，替换
				3. {rwx}：读，写，执行 ,分割可以同时设置多个权限
				4. mode=421：指定权限 r-4 w-2 x-1 即7：rwx 6：rw 1：r
				5. -R：递归更改
			3. 权限解释
				1. r 读权限 可以查看文件内容 可以列出目录中的内容
				2. w 写权限 可以修改文件内容 可以在目录中创建、删除文件
				3. x 可以执行文件 可以进入目录

		1. 是一个功能强大的全屏幕文本编辑器，是Linux/UNIX上最常用的文本编辑器，它的作用是建立、编辑、显示文本文件
		2. Vim没有菜单，只有命令
	2. Vim使用 命令模式、插入模式、编辑模式
		1. 插入命令
			1. a 在光标所在字符后插入
			2. A 在光标所在行的末尾插入
			3. i 在光标所在字符前插入
			4. I 在光标所在行行首插入
			5. o 在光标下插入新行
			6. O 在光标上插入新行
		2. 定位命令
			1. :set nu 设置行号
			2. :set nonu 取消行号
			3. gg或G到 到第一行或到最后一行
			4. nG 到第n行
			5. :n 到第n行
			6. $ 到行尾
			7. 0 到行首
		3. 删除命令
			1. x 删除光标所在处字符
			2. nx 删除光标所在处后n个字符
			3. dd 删除光标所在行，ndd删除n行
			4. dG 删除光标所在行到文件末尾内容
			5. D 删除光标所在处到行尾内容
			6. :n1，n2d 删除指定范围的行
		4. 复制和剪切命令
			1. yy 复制当前行
			2. nyy 复制当前行一下n行
			3. dd 剪切当前行
			4. ndd 剪切当前行一下n行
			5. p、P 粘贴在当前光标所在行下或行上
		5. 替换和取消命令
			1. r 取代光标所在处字符
			2. R 从光标所在处开始替换字符，按Esc结束
			3. u 取消上一步操作
		6. 搜索和搜索替换命令
			1. /string 搜索指定字符串，搜索时忽略大小写:set ic 、不忽略大小写 set noic
			2. n 搜索指定字符串的下一个出现位置
			3. :%s/old/new/g 全文替换指定字符串 /g换成/c则需要询问确认
			4. :n1,n2s/old/new/g 在一定范围内替换字符串
		7. 保存和退出
			1. :w 保存修改
			2. :W new_filename 另存为指定文件
			3. :wq 保存修改并退出
			4. ZZ 快捷键，保存修改并推出
			5. :q! 不保存修改退出
			6. :wq! 保存修改并退出（文件所有者及root可使用）
	3. Vim使用技巧
		1. 导入命令执行结果
			1. :r filename 导入文件
			2. :!命令 不退出Vim，查看执行命令
			3. :r !命令 不退出Vim，导入执行命令
			4. 定义快捷键 :map 快捷键 触发命令 
				1. map ^P I#<ESC>	^P:ctrl+vp 定义ctrl+p的快捷键 转换成注释
				2. map ^B 0x 转换成非注释
				3. map ^H i（加上插入内容） 快捷插入相关内容
				4. :1,4s/^/#/g  在第一行到第四行替换所有行的行首为#
				5. :1,$s/^#//g  在第一行到最后一行替换所有行的行首尾空
				6. ab 原文字 替换文字 定义快捷替换功能
			5. 插入/需要转义字符\
			6. .vimrc是vim的配置文件，可以配置编辑模式的命令

5. 权限管理
	1. ACL权限 Access Control List(访问控制列表)
		1. 简介与开启
			1. dumpe2fs -h 根分区文件系统名
				1. -h 进现实超级快中信息，而不显示磁盘块组的详细信息
				2. 查询指定分区详细文件系统信息命令
			2. df -h 查看文件分区
			3. mount -o remount,acl / 重新挂载 特殊挂载 临时赋予权限
			4. vi /etc/fstab （该命令是系统开启默认挂载的项目） 在defaults后加入,acl（现在的default都支持acl） 永久开启 mount -o remount / 重新挂载，或重启生效
		2. 查看与设定
			1. getfacl 文件名 查看acl权限 不支持绝对路径
			2. setfacl 选项 文件名
				1. -m 设定ACL权限
					1. setfacl -m u(给用户，g给组，m最大权限):rwx 目录
				2. -x 删除指定的ACL权限
				3. -b 删除所有的ACL权限
				4. -d 设定默认的ACL权限
				5. -k 删除默认的ACL权限
				6. -R 递归设定ACL权限
		3. 最大有效权限与删除ACL权限
			1. 需要与mask权限相与之后的权限才是真正能够拥有的权限
			2. 删除ACL权限
				1. setfacl -x u(g):用户名(组名) 文件名
				2. setfacl -b 文件名 会删除所有的ACL权限
		4. 默认与递归ACL权限
			1. setfacl -m u(给用户，g给组，m最大权限):rwx -R 目录 递归设定
			2. 默认ACL权限 d:u:用户名(组名):rwx 则设置为默认权限
	2. 文件特殊权限
		1. SetUID
			1. 功能
				1. 只有可以执行的二进制程序才能设定SUID权限
				2. 命令执行者要对该程序拥有x（执行）权限
				3. 命令执行者在执行该程序时获得该程序文件属主的身份（在执行程序的过程中灵魂附体为文件的属主）
				4. SetUID权限只在该程序执行过程中有效，也就是说身份改变只在程序执行过程中有效
			2. 即在执行时会暂时性的拥有超级管理员权限
			3. 例如passwd有SUID权限，但cat命令没有
			4. 设定SetUID的方法
				1. 4代表SUID，在权限前加4就代表设置了SUID
				2. 同理2代表SetGID、1代表Sticky BIT
				3. 可以用u+s的方式指定，或u-s的方式取消、
			5. 注意大S事没有执行权限的，需要先赋x权限
			6. 关键目录应严格控制写权限 例如/ 、 /usr
			7. 建议对系统中默认应该具有SetUID权限的文件作一列表，定时检查有没有这之外的文件被设置了SetUID权限
		2. SetGID
			1. 功能
				1. 针对目录
					1. 普通用户必须对此目录拥有r和x权限，才能进入此目录
					2. 普通用户在此目录中的有效组会变成此目录的属组
					3. 若普通用户对此目录拥有w权限时，新建的文件的默认属组是这个目录的属组
				2. 针对文件和SetUID相似
				3. 例如/usr/bin/locate 是搜索/var/lib/mlocate/mlocate.db
			2. 其他功能类似于SetUID
		3. Sticky BIT
			1. 粘着位目前只对目录有效
			2. 普通用户对该目录拥有w和x权限，即普通用户可以在此目录拥有写入权限
			3. 如果没有粘着位，因为普通用户拥有w权限，所以可以删除此目录下所有文件，包括其他用户建立的文件。一旦赋予了粘着位，除了root可以删除所有文件，普通用户就算拥有w权限，也只能删除自己建立的文件，但是不能删除其他用户建立的文件
	3. 文件系统属性chattr权限（能够限制root）
		1. 命令格式 chattr [+-=] [选项] 文件或目录名
			1. +、-、= 增加删除等于权限
			2. i 如果对文件设置i，那么不允许对文件进行删除、改名，也不能添加和修改数据；如果对目录设置i属性，那么只能修改目录下文件的数据，但不允许建立和删除文件
			3. a 如果对文件设置a属性，那么只能在文件中增加数据，但是不能删除也不能修改数据；如果对目录设置a属性，那么只允许在目录中建立和修改文件，但是不允许删除
		2. lsattr 选项 文件名
			1. -a 显示所有文件和目录
			2. -d 若目标是目录，仅列出目录本身的属性
	4. 系统命令权限：sudo权限
		1. 操作的对象是系统命令
		2. 将本来是超级用户能够执行的权限赋予普通用户
		3. 操作
			1. visudo 把命令赋予普通命令 修改/etc/sudoers文件
			2. root ALL=(ALL) ALL 用户 被管理主机的地址，这里注意不是来源ip=（可使用的身份） 授权命令（绝对路径）
			3. %wheel ALL=(ALL) ALL 给某一组用户授权
		4. sudo -l 查看管理员赋予的权限
6. 文件系统管理
	1. 分区和文件系统
		1. 分区
			1. 主分区：总共最多只能分四个
			2. 扩展分区：只能有一个，也算主分区的一种，在其中可以再分分区
			3. 逻辑分区：逻辑分区是在扩展在分区中划分的，如果是IDE硬盘，Linux最多支持59个逻辑分区，如果是SCSI硬盘Linux最多支持11个逻辑分区
			4. 挂载在/dev下
		2. 文件系统
			1. ext2：最大支持16TB的分区和最大2TB的文件
			2. ext3：带日志的ext2
			3. ext4：最大1EB文件系统和16TB的文件、无限量子目录、Extents连续数据块概念、多块分配、延迟分配等优势
	2. 文件系统常用命令
		1. df [选项] [挂载点] 显示分区信息
			1. -a 显示所有的文件系统信息，包括特殊文件系统，如/proc、/sysfs
			2. -h 使用习惯单位显示容量，如KB，MB或GB等
			3. -T 显示文件系统类型
			4. -m 以MB为单位显示容量
			5. -k 以KB为单位显示容量（默认单位即KB）
		2. du [选项] [目录或文件名] 统计目录或文件大小（但一般不用它统计文件大小，因为统计不是递归统计）
			1. -a 显示每个子文件的磁盘占用量 默认只统计子目录的磁盘占用量
			2. -h 使用习惯单位显示磁盘占用量
			3. -s 统计总占用量，二步列出子目录和子文件的占用量
			4. 注意该命令负载较高速度较慢
		3. df命令是从文件系统考虑的，不仅考虑文件占用，还要统计被命令、程序占用的空间。du命令流是面向文件的，只会计算文件或目录占用的空间
		3. fsck [选项] 分区设备文件名 文件系统修复命令
			1. -a 不用显示用户提示，自动修复文件系统
			2. -y 自动修复 有些文件只支持-y
			3. 开机自动启动
		4. dumpe2fs 分区设备名 显示磁盘状态
		5. 挂载命令
			1. 查询与自动挂载
				1. mount [-l] 查询系统中已经挂载的设备，-l会显示卷标名称
				2. mount -a 依据配置文件 /etc/fstab 的内容，自动挂载
			2. 要注意配置文件不能输入插拔设备，否则无法正常启动
			3. 挂载命令
				1. mount [-t 文件系统] [-L 卷标名] [-o 特殊选项] 设备文件名 挂载点
				2. -t文件系统：加入文件系统类型来指定挂载的类型，可以ext3、ext4、iso9660等文件系统
				3. -L卷标名：挂载指定卷标的分区，而不是安装设备文件名挂载
				4. -o特殊选项：可以指定挂载的额外选项
			4. 挂载光盘与U盘，一般在/mnt/目录下自己创建目录用来挂载相应设备
			5. umount 设备文件名或挂载点
		6. 注意Linux默认是不支持NTFS文件系统，挂载U盘时，fat是fat16，vfat是fat32，mount -t fat/vfat 文件系统 挂载目录
		7. NTFS的Linux支持
			1. 内核编译，即将驱动编入内核重新编译
			2. 第三方软件，例如NTFS-3G http://www.tuxera.com/community/ntfs-3g-download/
	3. fdisk分区
		1. fdisk -l 查看分区信息
		2. fdisk 文件系统 分区命令
		3. partprobe 刷新分区表
		4. mkfs 格式化
		5. 挂载
	4. 修改默认挂载项 /etc/fstab文件
		1. 分区设备文件名或UUID（硬盘通用唯一识别码）
		2. 挂载点
		3. 文件系统名称
		4. 挂载参数
		5. 制定分区是否被dump备份，0代表不备份，1代表每天备份，2代表不定期备份
		6. 只等分区是否被fsck监测，0代表不检测，其他数字代表检测优先级，1最大
	5. 当初始文件写错时，启动时会报错，故需要谨慎
